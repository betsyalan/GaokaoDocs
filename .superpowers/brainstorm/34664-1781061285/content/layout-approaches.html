📊 数据库集成 — 整体布局方案

现有系统：顶部导航 + 左侧文件列表(可伸缩) + 主内容区。数据库浏览如何融入？

A

导航 Tab 切换模式

NavBar 新增「数据」Tab，点击后左侧变为数据表选择器，右侧为数据浏览

  • 与文档浏览完全隔离，互不干扰
  • 导航路径清晰
  • 额外切换步骤
B

侧栏分类融入

文件侧栏新增「高考数据」分类，展开后显示各数据表，点击即加载数据视图

  • 无缝融入现有文件浏览流程
  • 操作路径短
  • 数据与文档混在一起，视觉上不够区分
C

混合方案（推荐）

NavBar 新增「数据」按钮，点击后侧栏变为数据表列表 + 筛选面板，主区显示数据表格。保留文件侧栏的独立身份

  • 视觉上清晰区分文档与数据
  • 侧栏可承载筛选控件
  • 符合用户说的"上部Tab,左侧列表"
  • 需要实现两套侧栏逻辑
